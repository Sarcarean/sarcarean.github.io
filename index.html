Description: so this project was to read a high-density non-vol memory device that is in a BGA-24 package and at 1.8V. To accomplish this, I used an Altera CPLD to implement a dual bank I/O with high-speed PIO in order to input high-speed signal from my MCU and output it as 1.8V. Slower designs could have gotten away with a level translator, but when you reach speeds of over 5MHz (per signal) you will surpass the ability of most translator ICs. For peripheral control, I used an ARM processor (Cortex-M3) with an integrated 480MHz PLL (needed to generate high-speed USB bulk transfers). I designed the PCB schematic in Eagle and made a 4-layer board. For PC side, I used LIBUSB for the Window's Driver and developed the Windows software in .net. For final testing, I set the SPI speed to 32MHz. (Hardware schematic)



Description: This is a demonstration of a typical development trifecta, that is a project that uses high-level desktop software (.net), firmware (C/C++), and programmable logic (VHDL). I just made this short video for my client (Cypress Semiconductor) showing implementation of their newest ultra fast non-vol memory product capable of 333MB/s data transfer. The protocol is comprised of a 8-bit parallel IO, clocked with a differential signal with 50% duty cycle and data being loaded at the rising and falling edges of the clock.
